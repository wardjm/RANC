# RANC Network IP

Vivado IP package for the RANC neuromorphic network. The IP instantiates a configurable grid of spiking cores connected by an on-chip packet-routing network and exposes an AXI4-Stream slave port for packet injection and configurable output capture.

## Generating the Vivado Project

Open Vivado, navigate to `Tools → Run Tcl Script...`, select `rancnetwork.tcl`, and run it. The project is created in `vivado_project/`.

## Parameters

| Parameter | Description |
|-----------|-------------|
| Grid Dimension X / Y | Number of cores along each axis |
| Output Core X / Y | Coordinates of the core whose spikes are forwarded to the output stream |
| Memory Filepath | Absolute path (forward slashes, trailing slash) to the directory containing the `csram_NNN.mem` and `tc_NNN.mem` files generated by `rancutils.emulation.output_for_streaming` |

All other parameters (neuron count, axon count, weight width, etc.) have sensible defaults that match the standard 256-neuron / 256-axon configuration.

## Unit Tests

MyHDL + iverilog testbenches are located under `src/simulations/`. Run them with:

```bash
cd src/simulations/tests   # or src/simulations/networks
pytest
```

The GitHub Actions workflow at `$REPO_ROOT/.github/workflows/hardware_unit_tests.yml` runs these automatically on each push.

## Using This IP in a Project

This IP is designed to be used as part of the streaming block design described in [`hardware/Docs/streaming/streaming.md`](../../Docs/streaming/streaming.md). The companion IP cores required are:

- **TickGenerator** — divides the system clock to produce a configurable tick signal and ARM interrupt
- **AXI-StreamPacketBuffer** — buffers outgoing spike packets and presents them as an AXI4-Stream to the DMA engine
